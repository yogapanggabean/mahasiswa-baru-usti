2.3 Jenkins sebagai Automation Server  
Jenkins adalah automation server open -source yang ditulis dalam bahasa Java, 
dirancang khusus untuk memfasilitasi proses Continuous Integration dan Continuous 
Delivery/Deployment. Jenkins pertama kali dikembangkan oleh Kohsuke Kawaguchi pada 
tahun 2004 den gan nama Hudson, kemudian berganti nama menjadi Jenkins pada tahun 2011 
setelah terjadi perpecahan dengan Oracle. Saat ini, Jenkins dikelola oleh komunitas open -
source yang besar dan aktif di bawah naungan Linux Foundation's Continuous Delivery 
Foundation (CDF).  
Sebagai automation server, Jenkins bertugas mengorkestrasikan seluruh pipeline 
CI/CD dengan cara mengotomatisasi proses build, menjalankan pengujian (unit test, 
integration test), serta memfasilitasi proses deployment ke berbagai lingkungan (development, 
staging, production). Penelitian oleh Abdel Mowad et al. (2022) menunjukkan bahwa 
penggunaan CI/CD seperti Jenkins secara signifikan mampu mempersempit kesenjangan 
antara tim developer dan tim operasi, yang merupakan inti dari filosofi DevOps.

2.4 Arsitektur  Jenkins  
Jenkins menggunakan arsitektur Master -Agent (sebelumnya dikenal sebagai Master -
Slave). Dalam arsitektur ini, Jenkins Master bertanggung jawab untuk menjadwalkan build job, 
mendistribusikan pekerjaan ke agent, memantau status agent, serta menyajikan hasil b uild 
melalui antarmuka web. Sementara itu, Jenkins Agent (Worker Node) adalah mesin yang 
secara aktif menjalankan proses build sesuai instruksi dari Master.  
Arsitektur ini memungkinkan Jenkins untuk menskalakan kapasitas build dengan 
mudah, karena beban pekerjaan dapat didistribusikan ke banyak agent yang berjalan di mesin 
yang berbeda, bahkan di sistem operasi yang berbeda secara bersamaan. Komunikasi antara 
Master dan Agent dilakukan melalui protokol Jenkins Remoting menggunakan JNLP atau 
SSH.

 3.5 Manajemen Plugin Jenkins  
Plugin adalah komponen yang memperluas fungsionalitas Jenkins. Pengelolaan plugin 
dilakukan melalui menu Manage Jenkins > Manage Plugins. Terdapat empat tab utama dalam 
halaman Plugin Manager, yaitu: Updates (plugin yang perlu diperbarui), Available (plugi n baru 
yang dapat dipasang), Installed (plugin yang sudah terpasang), dan Advanced (konfigurasi 
sumber plugin).  
Beberapa plugin esensial yang perlu dipasang untuk implementasi CI/CD yang lengkap 
antara lain: Git Plugin (integrasi dengan repository Git), Pipeline Plugin (mendukung pipeline 
as code), Maven Integration Plugin (untuk proyek berbasis Maven), Credentials Plugin 
(manajemen kredensial yang aman), Blue Ocean (antarmuka visual pipeline yang modern), dan 
Email Extension Plugin (notifikasi email).
